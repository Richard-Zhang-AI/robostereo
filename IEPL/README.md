# IEPL

> Imitative-Evolutionary Policy Learning

## 🧭 Overview

The core training pipeline for Imitative-Evolutionary Policy Learning (IEPL) is outlined below:

1. Utilize the initial frame of a MimicGen trajectory as the conditional input.
2. Generate a Cosmos video utilizing expert actions to serve as the reference rollout.
3. Predict actions using the current policy, followed by generating a policy rollout video via the Cosmos world model.
4. Compute the reconstruction discrepancy between the two generated videos as the reward signal to update the policy via GRPO.

The primary code entry points are located at:

- `iepl_cosmos/`: Implementations of the trainer, dataset, Cosmos worker, and reward logic.
- `scripts/cosmos/post_train_cosmos_rft.sh`: The standard training script.
- `scripts/cosmos/post_train_cosmos_rft_2gpu.sh`: The 2-GPU low-memory version.
- `scripts/docker_start_vla_cosmos.sh`: The recommended Docker launch script.

## 🚀 Training

### Standard Training

Execute the following commands within the container:

```bash
cd /workspace/IEPL
TASK_NAME=coffee \
VIDEO_ROOT=/workspace/datasets/train_openvla/coffee_128 \
COSMOS_CHECKPOINT_PATH=/workspace/checkpoints_files/coffee_action_conditioned/checkpoints-RGB/model_ema_fp32.pt \
COSMOS_NEGATIVE_PROMPT_FILE=/workspace/assets/action_conditioned/basic/coffee/inference_params.json \
bash scripts/cosmos/post_train_cosmos_rft.sh
```

### 2-GPU Low-Memory Training

## 🧪 Reward Construction

| Component | Source |
| --- | --- |
| Initial frame | The first frame of the MimicGen trajectory. |
| Expert actions | MimicGen annotations / action sequences. |
| Reference rollout | Video generated by Cosmos using `initial_frame + expert_actions`. |
| Policy rollout | Video generated by Cosmos using `initial_frame + policy_actions`. |
| Optimization target | Reconstruction-based reward computed between the two generated videos. |

In the current scripts, the reward is predominantly based on MSE, with the LPIPS weight disabled by default. For specific hyperparameters, please refer to `scripts/cosmos/post_train_cosmos_rft*.sh` and the configuration files in `iepl_cosmos/config/*.yaml`.
